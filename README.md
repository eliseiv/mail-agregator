# Mail Aggregator

Сервис-агрегатор электронной почты: один пользователь подключает несколько IMAP-почтовых ящиков (Gmail, Yandex, Mail.ru, Outlook и др.) и читает входящие сообщения в едином inbox, отвечает и пишет новые письма из веб-интерфейса.

## Назначение

- Закрытый сервис: вход только для пользователей, заведённых супер-админом.
- Каждый пользователь привязывает 1..N своих почтовых аккаунтов.
- Фоновый worker раз в 5 минут синхронизирует входящие со всех активных аккаунтов.
- Отправка ответов и новых писем — через SMTP-серверы привязанных аккаунтов.
- Хранение писем (заголовки + plain-text тело) в PostgreSQL, вложений — в MinIO. Ретенция 30 дней.

## Быстрый старт

См. [`docs/07-deployment.md`](./docs/07-deployment.md). Кратко:

1. Скопировать `.env.example` -> `.env`, заполнить секреты (см. таблицу env-переменных в `07-deployment.md`).
2. `docker compose up -d --build`.
3. При первом старте автоматически создаётся супер-админ из `ADMIN_LOGIN`/`ADMIN_PASSWORD`.
4. UI доступен на `http://localhost:8080/`.
