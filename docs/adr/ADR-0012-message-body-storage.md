# ADR-0012: Хранение тел писем — plain text only в PostgreSQL

- **Статус:** accepted
- **Дата:** 2026-05-05

## Context

ТЗ зафиксировало: показываем plain-text тело письма, отправляем plain-text. Это критично для безопасности UI (никакого HTML-исполнения, никаких трекинг-пикселей, никаких внешних ресурсов).

Что делать с письмами, в которых только HTML-часть? И где хранить тело — в БД или в MinIO?

## Decision

- Поле `messages.body_text TEXT NOT NULL` — нормализованное plain-text тело.
- Парсинг при сохранении (см. модуль `mail-sync` в `05-modules.md`):
  1. Если письмо имеет `text/plain` part — берём его.
  2. Если только `text/html` — конвертируем в plain text библиотекой `html2text` (>= 2024.2) с настройками: `body_width=0` (без переноса), `ignore_images=True`, `ignore_links=False` (но ссылки — как `[text](url)` markdown-стиль).
  3. Если ни того, ни другого (например, только attachments) — `body_text = ''`, флаг `body_present=false`.
- Поля `body_text` хранятся в PostgreSQL (TEXT). Размер тела ограничен 1 MiB; при превышении — обрезаем + флаг `body_truncated=true`.
- HTML-часть **не сохраняется**. Если в будущем понадобится "посмотреть оригинал HTML" — отдельный ADR + изменение модели данных.
- При отправке (модуль `mail-send`): пользователь пишет plain text -> отправляется как `text/plain; charset=utf-8` MIME-сообщение. Никакого HTML-альтернативы.

## Consequences

**Плюсы:**
- Безопасный UI — никакого XSS-вектора через HTML-почту.
- Простой fulltext-поиск в будущем (если понадобится — `tsvector` на `body_text`).
- Размер БД предсказуем (1 MiB max на письмо × 750к ≈ 750 GB max — но реально средний размер << 100 KiB).

**Минусы:**
- Письма с богатым форматированием (таблицы, изображения) теряют формат. Mitigation: пользователь видит ссылки и текст; для всего остального — оригинальный почтовый клиент.

## Alternatives considered

- **Хранить HTML и санитайзить через DOMPurify/bleach**: усложняет UI, увеличивает поверхность атаки. Отклонено.
- **Хранить тело в MinIO**: оверкилл; плюс это миллион операций find для inbox listing.
