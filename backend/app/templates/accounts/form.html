Как добавить почту?

Современные почтовые сервисы запрещают сторонним приложениям логин по обычному паролю. Нужен «пароль приложения» (app password) — одноразово сгенерированный длинный пароль, который ты вставляешь сюда вместо основного. Параметры серверов IMAP/SMTP подставятся автоматически для популярных доменов после ввода email.

Gmail (@gmail.com)
  1. Включи 2FA: myaccount.google.com/security → «2-Step Verification» → On.
  2. Включи IMAP: открой Gmail → «See all settings» → вкладка «Forwarding and POP/IMAP» → «Enable IMAP» → «Save Changes».
  3. Создай app password: myaccount.google.com/apppasswords → введи название → «Create» → скопируй 16-символьный пароль (показывается один раз).
  4. В форме: email you@gmail.com, пароль — этот 16-символьный (без пробелов). Хосты подставятся: imap.gmail.com:993 SSL, smtp.gmail.com:465 SSL.

Корпоративный Workspace Gmail: app passwords часто отключены администратором домена. Тогда сервис не сможет подключиться (OAuth не поддерживается).

Яндекс (@yandex.ru, @yandex.com, @ya.ru)
  1. Включи 2FA: id.yandex.ru/security → «Двухфакторная аутентификация».
  2. Включи IMAP в почте: mail.yandex.ru → «Все настройки» → раздел «Почтовые программы» → «С сервера imap.yandex.ru по протоколу IMAP» → «Сохранить изменения».
  3. Создай пароль приложения: на той же странице безопасности секция «Пароли приложений» → «Создать новый пароль» → выбери «Почта».
  4. В форме: email + сгенерированный пароль. Подставятся imap.yandex.ru:993 SSL, smtp.yandex.ru:465 SSL.
Mail.ru и семейство (@mail.ru, @inbox.ru, @bk.ru, @list.ru)
  1. Включи 2FA: account.mail.ru/user/2-step-auth → «Включить».
  2. Создай пароль для внешнего приложения: account.mail.ru/user/2-step-auth/passwords → «Добавить» → имя любое → подтверди текущим паролем → скопируй.
  3. IMAP включён по умолчанию — ничего больше делать не нужно.
  4. В форме: email + пароль приложения. Подставятся imap.mail.ru:993 SSL, smtp.mail.ru:465 SSL.
Outlook / Hotmail / Live (@outlook.com, @hotmail.com, @live.com)
  1. Включи 2FA: account.live.com/proofs/Manage → «Two-step verification».
  2. Создай app password: account.live.com/proofs/AppPassword → «Create a new app password».
  3. В форме: email + app password. Подставятся outlook.office365.com:993 SSL, smtp.office365.com:587 STARTTLS (важно — не SSL, а STARTTLS).

Корпоративный Office 365 (рабочий ящик через свой домен): app passwords часто запрещены — не подключится.

Apple iCloud (@icloud.com, @me.com, @mac.com)
  1. Включи 2FA для Apple ID: appleid.apple.com → «Sign-In and Security» → 2FA.
  2. Создай app-specific password: там же → «App-Specific Passwords» → «Generate Password» → метка любая → скопируй (формат xxxx-xxxx-xxxx-xxxx с дефисами).
  3. В форме параметры введи руками: imap.mail.me.com:993 SSL, smtp.mail.me.com:587 STARTTLS. В разделе «Использовать отдельные SMTP-учётные данные» укажи свой email как «SMTP-логин» (для iCloud это обязательно).
Yahoo Mail (@yahoo.com, @yahoo.ru, @ymail.com)
  1. Включи 2FA + создай app password: login.yahoo.com/account/security → «Two-step verification» → затем «Generate and manage app passwords» → «Generate».
  2. В форме руками: imap.mail.yahoo.com:993 SSL, smtp.mail.yahoo.com:465 SSL.
Другой провайдер / свой mail-сервер

Auto-suggest не подскажет — найди настройки на сайте провайдера (раздел «IMAP settings» / «настройка почтовой программы»). Чаще всего:

  • IMAP: imap.<домен>:993 SSL (или mail.<домен>)
  • SMTP: smtp.<домен>:465 SSL либо 587 STARTTLS
  • Логин — обычно полный email; в редких случаях нужен отдельный SMTP-логин — раздел «Использовать отдельные SMTP-учётные данные» ниже.

ProtonMail без платной подписки не работает: IMAP/SMTP открывается только через локальный ProtonMail Bridge, до которого Docker-контейнер не дотянется.

Что делать если «Проверить соединение» не работает
  • Application-specific password required (Gmail) — используешь обычный пароль, нужен app password.
  • imap_login_failed / Invalid credentials — пароль неверный, либо нужен app password.
  • imap_connect_error / smtp_connect_error — host/port/SSL не те. Сверь по таблице выше.
  • invalid_host — ввёл localhost или приватный IP, сервис их блокирует (защита от SSRF).
  • Web login required — провайдер требует первый раз войти в браузере (подтвердить «это вы»), затем повторить.
  • Тест проходит для IMAP, но падает для SMTP — попробуй раздел «Использовать отдельные SMTP-учётные данные» и укажи свой email как SMTP-логин.
